## 1. Project Overview

Design and verify a **synthesisable, parameterised AXI4 crossbar interconnect** in SystemVerilog. This is the central on-chip fabric for a RISC-V SoC, connecting multiple AXI4 masters (CPU, DMA, GPU) to multiple AXI4 slaves (memory controller, peripherals, GPIO). The crossbar provides independent read and write routing with per-channel arbitration, outstanding transaction tracking, configurable address decoding, and error responses for unmapped regions.

### Key features

| Feature | Detail |
|---|---|
| Masters | Parameterised, default `N_MASTERS = 2` |
| Slaves | Parameterised, default `N_SLAVES = 4` |
| Data width | 32 bits |
| Address width | 32 bits |
| ID width | 4 bits (master-side), extended to `4 + $clog2(N_MASTERS)` bits on slave-side |
| Burst types | FIXED, INCR (WRAP not required) |
| Burst length | 1–256 beats (AXI4 full) |
| Arbitration | Round-robin (default), configurable fixed-priority |
| Outstanding txns | Multiple per master, tracked via ID field extension |
| Write interleaving | Not supported — write path locked per master until WLAST |
| Unmapped address | DECERR generated by internal error slave |
| Backpressure | Full propagation without cross-blocking |
| Clock | Single clock domain, synchronous active-high reset (`srst`) |

## 2. Architecture Specification

### 2.1 AXI4 Signal Definitions

All interfaces follow the ARM AMBA AXI4 specification. Signal names use the standard AXI4 naming convention with `_m` suffix for master-facing ports and `_s` suffix for slave-facing ports where disambiguation is needed.

#### 2.1.1 Write Address Channel (AW)

| Signal | Width | Direction (Master→Slave) | Description |
|---|---|---|---|
| `awid` | `ID_W` | M→S | Write transaction ID |
| `awaddr` | `ADDR_W` | M→S | Write address |
| `awlen` | 8 | M→S | Burst length − 1 (0 = 1 beat, 255 = 256 beats) |
| `awsize` | 3 | M→S | Bytes per beat = 2^awsize |
| `awburst` | 2 | M→S | Burst type: 2'b00 = FIXED, 2'b01 = INCR |
| `awvalid` | 1 | M→S | Address valid |
| `awready` | 1 | S→M | Address ready |

#### 2.1.2 Write Data Channel (W)

| Signal | Width | Direction (Master→Slave) | Description |
|---|---|---|---|
| `wdata` | `DATA_W` | M→S | Write data |
| `wstrb` | `DATA_W/8` | M→S | Byte-lane strobes |
| `wlast` | 1 | M→S | Last beat in burst |
| `wvalid` | 1 | M→S | Data valid |
| `wready` | 1 | S→M | Data ready |

#### 2.1.3 Write Response Channel (B)

| Signal | Width | Direction (Slave→Master) | Description |
|---|---|---|---|
| `bid` | `ID_W` | S→M | Response transaction ID |
| `bresp` | 2 | S→M | Response: 2'b00 = OKAY, 2'b11 = DECERR |
| `bvalid` | 1 | S→M | Response valid |
| `bready` | 1 | M→S | Response ready |

#### 2.1.4 Read Address Channel (AR)

| Signal | Width | Direction (Master→Slave) | Description |
|---|---|---|---|
| `arid` | `ID_W` | M→S | Read transaction ID |
| `araddr` | `ADDR_W` | M→S | Read address |
| `arlen` | 8 | M→S | Burst length − 1 |
| `arsize` | 3 | M→S | Bytes per beat = 2^arsize |
| `arburst` | 2 | M→S | Burst type |
| `arvalid` | 1 | M→S | Address valid |
| `arready` | 1 | S→M | Address ready |

#### 2.1.5 Read Data Channel (R)

| Signal | Width | Direction (Slave→Master) | Description |
|---|---|---|---|
| `rid` | `ID_W` | S→M | Read data ID |
| `rdata` | `DATA_W` | S→M | Read data |
| `rresp` | 2 | S→M | Response |
| `rlast` | 1 | S→M | Last beat in burst |
| `rvalid` | 1 | S→M | Data valid |
| `rready` | 1 | M→S | Data ready |

### 2.2 ID Width Extension for Response Routing

The crossbar prepends the **master index** to the transaction ID before forwarding to slaves. This allows responses to be routed back to the originating master without maintaining lookup tables.

```
Master-side ID:  [ID_W-1 : 0]                        (4 bits default)
Slave-side ID:   [$clog2(N_MASTERS)+ID_W-1 : 0]      (5 bits default for 2 masters)

Slave-side ID = { master_index[$clog2(N_MASTERS)-1:0], master_id[ID_W-1:0] }
```

When a response arrives from a slave:
1. Extract `master_index` from the upper bits of the response ID.
2. Route the response to that master.
3. Strip the upper bits before presenting the response ID to the master.

### 2.3 Address Map Configuration

The address map is configured via parameter arrays in `axi_xbar_pkg.sv`:

```systemverilog
parameter logic [ADDR_W-1:0] SLAVE_BASE [N_SLAVES] = '{
    32'h0000_0000,   // Slave 0: Boot ROM / SRAM
    32'h1000_0000,   // Slave 1: Peripheral bus
    32'h2000_0000,   // Slave 2: External memory controller
    32'h4000_0000    // Slave 3: GPIO / misc
};

parameter logic [ADDR_W-1:0] SLAVE_MASK [N_SLAVES] = '{
    32'h0FFF_FFFF,   // Slave 0: 256 MB
    32'h0FFF_FFFF,   // Slave 1: 256 MB
    32'h1FFF_FFFF,   // Slave 2: 512 MB
    32'h3FFF_FFFF    // Slave 3: 1 GB
};
```

Address decode rule: `(addr & ~SLAVE_MASK[i]) == SLAVE_BASE[i]`

Equivalently: a transaction matches slave `i` when `(addr >= SLAVE_BASE[i]) && (addr <= SLAVE_BASE[i] + SLAVE_MASK[i])`.

If no slave matches, the transaction routes to the internal error slave which returns DECERR.

#### 2.5.2 `axi_addr_decoder.sv` — Address Decoder

**Purely combinational.** Takes an address, outputs the slave index and a `valid` flag.

Ports:
| Port | Width | Dir | Description |
|---|---|---|---|
| `addr` | `ADDR_W` | in | Address to decode |
| `slave_idx` | `$clog2(N_SLAVES+1)` | out | Decoded slave index (N_SLAVES = error slave) |
| `addr_valid` | 1 | out | `1` if any slave matched, `0` for unmapped |

Logic:
- Priority-encoded scan over slaves 0 to N_SLAVES−1.
- First match wins (address ranges must not overlap — this is a configuration constraint, not enforced in hardware).
- If no match: `slave_idx = N_SLAVES` (points to error slave), `addr_valid = 0`.
- **Use `always @(*)` for this module** (combinational block reading parameters — avoid `always_comb` for iverilog compatibility).

#### 2.5.3 `axi_arbiter.sv` — Round-Robin / Fixed-Priority Arbiter

Parameterised N-input arbiter. Used for both read and write address channel arbitration.

Parameters:
| Parameter | Default | Description |
|---|---|---|
| `N_REQ` | 2 | Number of requestors |
| `MODE` | `ARB_ROUND_ROBIN` | Arbitration mode |

Ports:
| Port | Width | Dir | Description |
|---|---|---|---|
| `clk` | 1 | in | Clock |
| `srst` | 1 | in | Synchronous reset (active-high) |
| `req` | `N_REQ` | in | Request vector (one-hot: bit i = master i requesting) |
| `grant` | `N_REQ` | out | Grant vector (one-hot: at most one bit set) |
| `grant_idx` | `$clog2(N_REQ)` | out | Index of granted requestor |
| `grant_valid` | 1 | out | `1` if any grant issued |
| `lock` | 1 | in | While asserted, hold current grant (used for write bursts) |

**Round-robin FSM:**

State: `last_grant` register (index of last granted master, `$clog2(N_REQ)` bits wide).

Combinational logic:
1. If `lock` is asserted and current grantee still has `req` high → maintain grant.
2. Otherwise, scan from `last_grant + 1` wrapping around. First `req[i]` that is high gets the grant.
3. On grant handshake completion (when the downstream valid/ready fires), update `last_grant`.

Sequential block:
```
always_ff @(posedge clk)
    if (srst)
        last_grant <= '0;
    else if (grant_valid && !lock)
        last_grant <= grant_idx;
```

**Fixed-priority mode:** Grant to lowest-indexed active requestor. `last_grant` is unused.

#### 2.5.4 `axi_w_path.sv` — Write Path

Handles the full write transaction: AW (address), W (data), and B (response) channels for all masters and slaves.

Parameters: `N_MASTERS`, `N_SLAVES`, all widths from package.

Ports: Arrays of AW/W/B signals — one set per master, one set per slave, plus error slave interface.

**Internal structure:**

```
                        ┌────────────────┐
  Master 0 AW ────────►│                │────────► Slave 0 AW
  Master 1 AW ────────►│  AW Arbiter    │────────► Slave 1 AW
                        │  + Decoder     │────────► Slave 2 AW
                        │  + Demux       │────────► Slave 3 AW
                        └────────────────┘────────► Err Slave AW
                               │
                               │ grant_idx → selects W path
                               ▼
                        ┌────────────────┐
  Master 0 W  ────────►│  W Switch      │────────► Slave 0 W
  Master 1 W  ────────►│  (locked until  │────────► Slave 1 W
                        │   WLAST)       │────────► Slave 2 W
                        └────────────────┘────────► Slave 3 W
                                                ──► Err Slave W

                        ┌────────────────┐
  Master 0 B  ◄────────│  B Return Mux  │◄──────── Slave 0 B
  Master 1 B  ◄────────│  (route by     │◄──────── Slave 1 B
                        │   bid upper    │◄──────── Slave 2 B
                        │   bits)        │◄──────── Slave 3 B
                        └────────────────┘◄──────── Err Slave B
```

**AW Arbitration and Routing:**

For each slave (including error slave), there is one instance of `axi_arbiter` that arbitrates among all masters whose decoded address points to that slave. Implementation approach:

1. Each master's AW channel is decoded by an `axi_addr_decoder` to determine the target slave.
2. For each slave `s`, build a request vector: `aw_req[s][m] = m_awvalid[m] && (decoded_slave[m] == s)`.
3. Arbiter for slave `s` grants one master → that master's AW is connected to slave `s`.
4. The AW handshake (`awvalid` & `awready`) completes, locking the W path.

**W Path Locking:**

When an AW handshake completes for master `m` → slave `s`:
1. Record the mapping: `w_route[m] = s` (register).
2. Assert `lock` on the AW arbiter for slave `s` — no new AW grants to this slave from other masters until the W burst completes.
3. Route master `m`'s W channel to slave `s`.
4. When `wvalid && wready && wlast` fires, release the lock. Clear `w_route[m]`.

State machine per master (write tracking):
```
typedef enum logic [1:0] {
    W_IDLE    = 2'b00,   // No active write
    W_ADDR    = 2'b01,   // AW accepted, waiting for W data
    W_DATA    = 2'b10,   // Streaming W data beats
    W_RESP    = 2'b11    // Waiting for B response
} w_state_t;
```

Transitions:
- `W_IDLE → W_ADDR`: AW valid & AW ready
- `W_ADDR → W_DATA`: First W beat valid & ready (or `W_ADDR → W_RESP` if `awlen == 0` and single-beat completes immediately)
- `W_DATA → W_RESP`: `wvalid && wready && wlast`
- `W_RESP → W_IDLE`: `bvalid && bready`

**B Response Routing:**

Responses from slaves carry the extended ID (`SID_W` bits). The upper `MSTR_IDX_W` bits identify the originating master. The B mux:

1. For each slave `s`, check if `s_bvalid[s]` is asserted.
2. Extract `master_idx = s_bid[s][SID_W-1 -: MSTR_IDX_W]`.
3. Route `s_bresp`, `s_bvalid` to master `master_idx`.
4. Strip upper bits: `m_bid[master_idx] = s_bid[s][ID_W-1:0]`.
5. Route `m_bready[master_idx]` back to slave `s`.
6. If multiple slaves have responses for the same master simultaneously, use round-robin among them (instantiate another arbiter or simple priority).

#### 2.5.5 `axi_r_path.sv` — Read Path

Handles AR (address) and R (data) channels.

**Structure mirrors write path but is simpler** — no locking required since read data is inherently ordered per ID and there is no separate data phase to lock.

```
                        ┌────────────────┐
  Master 0 AR ────────►│  AR Arbiter    │────────► Slave 0 AR
  Master 1 AR ────────►│  + Decoder     │────────► Slave 1 AR
                        │  + Demux       │────────► Slave 2 AR
                        └────────────────┘────────► Slave 3 AR
                                                ──► Err Slave AR

                        ┌────────────────┐
  Master 0 R  ◄────────│  R Return Mux  │◄──────── Slave 0 R
  Master 1 R  ◄────────│  (route by     │◄──────── Slave 1 R
                        │   rid upper    │◄──────── Slave 2 R
                        │   bits)        │◄──────── Slave 3 R
                        └────────────────┘◄──────── Err Slave R
```

**AR Arbitration:** Same pattern as AW — per-slave arbiter, grant one master at a time.

**R Response Routing:**
1. For each slave with `s_rvalid[s]` asserted, extract `master_idx` from upper bits of `s_rid`.
2. Route read data to the correct master.
3. Strip upper ID bits before presenting to master.
4. Handle `rlast` correctly — the R mux must stay connected to one slave for an entire burst (all beats until `rlast`). Use a simple lock: once an R beat is accepted from slave `s` for master `m`, keep that path connected until `rlast`.

**Outstanding read tracking per master:**

Each master may have multiple outstanding reads to different slaves. To route responses correctly, the R return mux must handle interleaved responses. Implementation:

- For each master, maintain a small FIFO (depth = max outstanding transactions, e.g., 4) recording which slave each outstanding read went to. When the AR handshake completes, push `slave_idx`. When `rvalid && rready && rlast`, pop.
- **Alternative (simpler, preferred):** Rely solely on the extended ID bits in the response — no FIFO needed. The slave-side ID already encodes the master index. Each slave's R channel is checked: if `rid` upper bits match master `m`, route to master `m`. Arbitrate if multiple slaves respond to the same master simultaneously.

Use the simpler ID-based approach (no FIFOs for read tracking).

#### 2.5.6 `axi_err_slave.sv` — Error Slave

A minimal AXI4 slave that accepts any transaction and responds with DECERR.

Ports: Single AXI4 slave interface (using `SID_W`-wide IDs).

**Write path FSM:**
```
typedef enum logic [1:0] {
    ERR_W_IDLE   = 2'b00,
    ERR_W_DATA   = 2'b01,   // Sink W beats until WLAST
    ERR_W_RESP   = 2'b10    // Drive B with DECERR
} err_w_state_t;
```
- `ERR_W_IDLE`: `awready = 1`. On `awvalid && awready`, capture `awid` → `ERR_W_DATA`.
- `ERR_W_DATA`: `wready = 1`. Sink all W beats. On `wlast && wvalid && wready` → `ERR_W_RESP`.
- `ERR_W_RESP`: Drive `bvalid = 1`, `bresp = RESP_DECERR`, `bid = captured_id`. On `bready` → `ERR_W_IDLE`.

**Read path FSM:**
```
typedef enum logic [1:0] {
    ERR_R_IDLE   = 2'b00,
    ERR_R_DATA   = 2'b01    // Drive R beats with DECERR until rlast
} err_r_state_t;
```
- `ERR_R_IDLE`: `arready = 1`. On `arvalid && arready`, capture `arid`, `arlen` → `ERR_R_DATA`, load beat counter.
- `ERR_R_DATA`: Drive `rvalid = 1`, `rresp = RESP_DECERR`, `rdata = 32'hDEAD_BEEF`, `rid = captured_id`. Decrement beat counter on `rready`. Assert `rlast` when counter reaches 0. On `rlast && rvalid && rready` → `ERR_R_IDLE`.

## 3. Coding Conventions

These conventions are **mandatory** and must be followed exactly in every file.

### 3.1 Language & Tools

- **Language**: SystemVerilog (`.sv`), targeting `iverilog -g2012`.
- **Use `always @(*)` instead of `always_comb`** for any combinational block that reads signals driven by submodule outputs (avoids iverilog infinite re-evaluation loops). Use `always_comb` only for purely local combinational logic within a module that does not read submodule outputs.
- `always_ff`, `logic`, `typedef enum`, packages are all fine.
- **No vendor-specific primitives.** Must simulate cleanly in iverilog + GTKWave.

### 3.2 Reset

- **Synchronous reset**, active-high, signal name `srst`.
- Every `always_ff` block: `always_ff @(posedge clk)` with `if (srst)` as the **first branch**.

### 3.3 FSM Pattern

```systemverilog
typedef enum logic [N:0] { STATE_A, STATE_B, ... } state_t;
state_t state, state_next;

always_ff @(posedge clk)
    if (srst)
        state <= STATE_A;
    else
        state <= state_next;

always @(*) begin     // or always_comb if no submodule reads
    state_next = state;  // default: hold
    // output defaults
    case (state)
        STATE_A: begin ... end
        STATE_B: begin ... end
        default: state_next = STATE_A;
    endcase
end
```

### 3.4 Handshake

- All inter-module and external interfaces use `valid`/`ready` handshake.
- A transfer occurs when `valid && ready` are both high on the same clock edge.
- A source must not deassert `valid` once asserted until `ready` is seen (AXI4 rule).

### 3.5 Naming

- `snake_case` for all signals, modules, files.
- Prefix testbench files with `tb_` (e.g., `tb_axi_arbiter.sv`).
- Prefix CocoTB test files with `test_` (e.g., `test_axi_arbiter.py`).
- Module names must match filenames exactly (minus `.sv`).

### 3.6 Author Line

First line of **every** source file (RTL, testbench, script, Python):

```
// Brendan Lynskey 2025
```

For Python files: `# Brendan Lynskey 2025`
For Markdown: `<!-- Brendan Lynskey 2025 -->`

### 3.7 Port Declaration Style

Use ANSI-style port declarations. One signal per line. Group by channel with comments:

```systemverilog
module axi_err_slave
    import axi_xbar_pkg::*;
(
    input  logic                clk,
    input  logic                srst,

    // AW channel
    input  logic                awvalid,
    output logic                awready,
    input  logic [SID_W-1:0]    awid,
    input  logic [ADDR_W-1:0]   awaddr,
    input  logic [7:0]          awlen,
    // ... etc
);
```

### 3.8 Parameterisation

Use `parameter` or `localparam` at module level. Import shared parameters from `axi_xbar_pkg`. Override at instantiation site where appropriate.

## 5. Implementation Order

**Build bottom-up. Verify each module in isolation before integrating.**

### Phase 1: Foundation modules

1. **`axi_xbar_pkg.sv`** — Write the package first. All other modules import it.
2. **`axi_addr_decoder.sv`** — Purely combinational. Write + verify immediately.

### 6.4 Slave BFM for Integration Tests

For integration-level SV and CocoTB tests, you need a **simple AXI4 slave behavioural model** that:
- Accepts write transactions and stores data in an associative array (`logic [DATA_W-1:0] mem [logic [ADDR_W-1:0]]`).

- Supports configurable backpressure (insert random `ready` deassertions).
- Responds with OKAY.

Place this in `tb/sv/axi_slave_bfm.sv` and `tb/cocotb/axi_slave_bfm.py` (or use cocotb-bus AXI4 classes if available, but a simple custom BFM is fine).

- Issue write and read transactions with configurable address, data, burst length, ID.
- Check response codes.
- Operate concurrently (two master BFMs running in parallel).

Place in `tb/sv/axi_master_bfm.sv` and implement in Python for CocoTB.

## 10. Stretch Goals

Implement these only after the core design passes all 109+ tests.

### 10.1 QoS (Quality of Service)

Add a 4-bit QoS field to AR and AW channels. The arbiter can use QoS to implement weighted round-robin: higher QoS → more grant slots.

### 10.2 AXI4-Lite Configuration Port

Add an AXI4-Lite slave port on the crossbar itself for runtime configuration:
- Read/write the address map (base + mask per slave).
- Read arbitration mode.
- Read transaction counters.

Register map:
| Offset | Name | Description |
|---|---|---|
| `0x00` | `SLAVE0_BASE` | Slave 0 base address |
| `0x04` | `SLAVE0_MASK` | Slave 0 address mask |
| `0x08` | `SLAVE1_BASE` | ... |
| ... | ... | ... |
| `0x40` | `ARB_MODE` | Arbitration mode (0 = RR, 1 = fixed) |
| `0x44` | `TXN_COUNT_W` | Total write transactions |
| `0x48` | `TXN_COUNT_R` | Total read transactions |

### 10.3 Transaction Reordering

Allow out-of-order read responses from different IDs. This requires per-ID tracking and reorder buffers. Significantly increases complexity.

### 10.4 Bandwidth Monitoring

Add per-master and per-slave beat counters. Expose via the AXI4-Lite configuration port or as top-level output ports.

### 10.5 AXI4 WRAP Burst Support

Add support for WRAP burst type (2'b10). Requires address wrapping logic in the address decoder and in the slave BFM.
