## ID Extension

```
Master-side:  [ID_W-1:0]           = 4 bits
Slave-side:   [SID_W-1:0]          = 5 bits (for 2 masters)

Encoding:     { master_index[0], original_id[3:0] }

Response routing: Extract master_index from upper bit(s) of response ID,
                  route to that master, strip upper bits.
```

## Address Map

| Slave | Base         | End          | Size   |
|-------|-------------|--------------|--------|
| 0     | 0x0000_0000 | 0x0FFF_FFFF | 256 MB |
| 1     | 0x1000_0000 | 0x1FFF_FFFF | 256 MB |
| 2     | 0x2000_0000 | 0x3FFF_FFFF | 512 MB |
| 3     | 0x4000_0000 | 0x7FFF_FFFF | 1 GB   |
| Error | unmapped    | unmapped     | вҖ”      |

Decode rule: `(addr & ~SLAVE_MASK[i]) == SLAVE_BASE[i]`

## Arbitration

- Per-slave round-robin arbiter for AW and AR channels
- Write path locks arbiter during W burst (until WLAST)
- Read path has no locking (AR requests are independent)
- Responses routed by ID extension вҖ” no lookup tables needed
